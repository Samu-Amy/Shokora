Run backend:
- cd /backend/
  - "go run cmd/app/main.go"


Run frontend:
- cd /frontend/
  - "npm run dev"
  - !non più utilizzabile (serve proxy) "npm run build" -> "npx serve build" (nginx-like)


Run Docker:
- "docker compose -f compose[.dev].yaml up"

Access DB in docker:
- "docker compose -f compose.dev.yaml exec db psql -U {postgres_user} -d {postgres_db}"

Access DB outside docker:
- "docker run --rm -it postgres:17 psql "host=host.docker.internal port=<port> user=<user> dbname=<db> sslmode=disable"

DB Migrations:
- create: "goose create <migration_name> sql"
- execute: "goose <driver (postgres)> <db_string ($1)> up"
- revert by 1: "goose <driver (postgres)> <db_string ($1)> down"
- revert to version: "goose <driver (postgres)> <db_string ($1)> down-to <version>"
- revert to version 0: "goose <driver (postgres)> <db_string ($1)> reset"

Nota: la versione è quella nel nome della migration (è un timestamp, quindi da <version> o fino a <version> esegue/revert tutte quelle prima/dopo)

to avoid passing the connection string, we can define:
- "export GOOSE_DRIVER=<driver>"
- "export GOOSE_DBSTRING=<conn_str>"


DB strings formats:
- $1 "postgresql://[user[:password]@][host][:port][/dbname][?param1=value1&param2=value2]"
- $2 "host=localhost port=5432 dbname=mydatabase user=myuser password=mypassword"

In this case:
- "postgresql://<user>:<password>@localhost:<port>/<db_name>?sslmode=<sll_mode (disable)>"
- "host=localhost port=<port> user=<user> password=<password> dbname=<db_name> sslmode=<ssl_mode>"
